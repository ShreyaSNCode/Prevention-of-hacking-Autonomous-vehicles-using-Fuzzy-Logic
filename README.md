# Prevention of Hacking Autonomous Vehicles using Fuzzy Logic

## 📌 Overview
Autonomous vehicles are vulnerable to cyberattacks that can compromise safety and reliability. This project applies **fuzzy logic** to detect and prevent hacking attempts by analyzing intrusion patterns, abnormal behaviors, and suspicious data flows. The system integrates machine learning concepts with rule‑based fuzzy inference to provide intelligent defense strategies.

## 🎯 Objectives
- Enhance cybersecurity in autonomous vehicles.
- Detect intrusion attempts using fuzzy logic rules.
- Simulate attack scenarios and evaluate system resilience.
- Provide a scalable framework for real‑world deployment.

## 🧠 Why Fuzzy Logic?
Unlike binary decision systems, fuzzy logic allows reasoning with uncertainty and partial truths. This makes it ideal for intrusion detection, where attack signals may be ambiguous. By defining fuzzy sets (e.g., *low risk*, *medium risk*, *high risk*) and inference rules, the system can:
- Classify suspicious activity more flexibly.
- Reduce false positives compared to rigid threshold systems.
- Adapt to evolving attack patterns.

## ⚙️ Tech Stack
- **Python** (Flask, SQLAlchemy, Pandas, NumPy)
- **Fuzzy Logic Frameworks** (scikit‑fuzzy)
- **Dataset**: Network intrusion dataset (sample of 250 rows)
- **Visualization**: Matplotlib, Seaborn
- **Environment**: VS Code
